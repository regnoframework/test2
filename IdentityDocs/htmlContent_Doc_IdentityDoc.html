Identity Document - IdentityDoc

The IdentityDoc document is an optional identity document that allows users to define ownership and include identifying markers for a collection of documents. If used, the IdentityDoc document can be referenced by all other documents.

Field Data Type Required Description
id String Yes A unique identifier that is referenced by all other documents that are associated with this IdentityDoc document. The proposed method of generation of this value is described in the section Unique Id Generation.
tags Array [String, String] No An array of key/pair free form string values. Used to store meta data related to the ownership of the associated documents.
type Fixed String Yes The fixed value is always set to IdentityDoc.

The IdentityDoc is a document of importance within the Regno Standard. This document though optional can have its use mandated by organising bodies to allow the body to identify the source owner of the document. It is intended that organising bodies could create their own IdentityDoc document and then supply the generated id value to end users who would be mandated to include the supplied id in each of the mandated documents.

The IdentityDoc document can also be used internally within organisations as a high level method of data rights access. An example of such use would be the creation of key/pair tag values that correspond with specific access rights. The user API implementation would use these document rights to only allow access to a given set of documents for a given set of users. Multiple IdentityDoc documents can be created and referenced by each of the other document types to create a powerful hierarchal ownership and access rights tree.
